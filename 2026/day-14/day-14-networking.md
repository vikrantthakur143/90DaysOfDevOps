| OSI Model (7 Layers)   | TCP/IP Model (4 Layers) | Purpose                                       | Example                                       |
| ---------------------- | ----------------------- | --------------------------------------------- | --------------------------------------------- |
| Layer 7 – Application  | Application             | User-facing protocols and services            | HTTP, HTTPS, FTP, DNS             |
| Layer 6 – Presentation | Application             | Data formatting, encryption, compression      | Data formatting, Encryption, Compression => SSL/TLS encryption, JPEG image formats, UTF-8 text encoding      |
| Layer 5 – Session      | Application             | Session establishment and management          | Establish session, Maintain session, Terminate session |
| Layer 4 – Transport    | Transport               | End-to-end communication (TCP, UDP)           | TCP, UDP => Reliability, Flow ontrol, Error checking, Port numbers             |
| Layer 3 – Network      | Internet                | Routing and IP addressing                     | Protocol: IP, ICMP.  Devices: Routes             |
| Layer 2 – Data Link    | Network Access          | Local network communication (Ethernet, Wi-Fi) | MAC addresses, Ethernet frames. Devices Switch             |
| Layer 1 – Physical     | Network Access          | Physical transmission of bits                 | Network cables, Fiber optics, Wi-Fi radio signals. Device: Hubs, Network cards, Cables            |

The OSI model is mainly a conceptual framework used for learning and troubleshooting networks. It breaks networking into seven detailed layers, making it easier to identify where a problem occurs.

What Are OSI and TCP/IP Models?
Both the OSI model and the TCP/IP model are frameworks that explain how data travels from one device to another over a network.
